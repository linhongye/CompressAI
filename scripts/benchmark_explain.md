# Benchmark Report JSON 字段说明

运行 `benchmark.py` 后，每个模型/质量等级会在 `report/` 目录下生成一个 JSON 文件。
以下以 `cheng2020-attn-mse-q1.json` 为例逐字段说明。

---

## 顶层元信息字段

| 字段 | 含义 |
|------|------|
| `name` | 报告标识符，由 `架构-metric-q级别` 组成 |
| `source` | 权重来源，`pretrained` = 使用官方预训练权重，`custom` = 自训练 |
| `architecture` | 模型架构名称 |
| `metric` | 训练时优化的失真度量，`mse` = 均方误差，`ms-ssim` = 多尺度结构相似度 |
| `dataset` | 测试集路径 |
| `num_images` | 参与测试的图片数量 |
| `compressed_dir` | 压缩后码流（`.bin`）的存放目录 |
| `reconstruction_dir` | 解压还原图（`.png`）的存放目录 |

### `metric` 字段的影响

- `"metric": "mse"` — 模型以最小化像素级均方误差为目标训练，倾向于获得较高的 PSNR，画面偏平滑。
- `"metric": "ms-ssim"` — 模型以最大化感知结构相似度为目标训练，倾向于保留纹理细节，MS-SSIM 更高，但 PSNR 可能略低于 mse 版本。

---

## `results` 字段（全数据集均值）

### `psnr-rgb`（单位：dB，越高越好）

**峰值信噪比**（Peak Signal-to-Noise Ratio），衡量重建图像与原图的像素级保真度。

公式：PSNR = 10 · log₁₀(MAX² / MSE)，其中 MAX = 255（8-bit 图像）。

| 范围 | 质量评价 |
|------|----------|
| < 30 dB | 差，肉眼可见明显失真 |
| 30 – 35 dB | 一般，有可见压缩伪影 |
| 35 – 40 dB | 良好，细节基本保留 |
| 40 – 45 dB | 优秀，接近无损视觉效果 |
| > 45 dB | 极优，几乎与原图无异 |

**本报告值：39.46 dB** — 属于**良好**范围。考虑到 q1 是压缩率最高（比特率最低）的质量等级，在如此低的比特率下仍能达到约 39.5 dB，说明 cheng2020-attn 模型压缩效率相当高。

> 注意：PSNR 对数据内容敏感。均匀区域（天空、海洋、裸地）的 PSNR 天然偏高；纹理复杂区域（城区、植被）偏低。本测试集为遥感影像切片，部分平滑瓦片使整体均值偏高，不同图片之间差异悬殊（见 `per_image` 范围 37 ~ 42.7 dB）。

---

### `mse`（均方误差，越低越好，仅作训练目标，不直接出现在 results 中）

**均方误差**（Mean Squared Error）是像素级重建误差的平方均值：

MSE = (1 / N) · Σ (原图像素 − 重建像素)²

MSE 本身不直接出现在报告的 `results` 字段，但它是 PSNR 的计算基础（PSNR = 10 · log₁₀(255² / MSE)），两者严格单调对应：**MSE 越低 ↔ PSNR 越高**。

| MSE 范围（8-bit 图像） | 对应 PSNR | 感知质量 |
|------------------------|-----------|----------|
| < 1 | > 48 dB | 极优，肉眼无法区分 |
| 1 – 10 | 40 – 48 dB | 优秀 |
| 10 – 100 | 30 – 40 dB | 一般到良好 |
| > 100 | < 30 dB | 差，明显失真 |

**本报告对应值：** PSNR 39.46 dB → MSE ≈ **7.4**（良好范围内）。

> 注意：MSE 作为损失函数的局限性在于它对每个像素一视同仁，会导致模型倾向于产生整体偏模糊的重建图（高频纹理细节被平滑掉），这也是为什么感知质量更好的模型通常会结合 MS-SSIM 或感知损失（perceptual loss）来训练。

---

### `ms-ssim-rgb`（范围 0 ~ 1，越接近 1 越好）

**多尺度结构相似度**（Multi-Scale Structural Similarity Index），比 PSNR 更接近人眼感知，同时考量亮度、对比度与结构信息。

| 范围 | 质量评价 |
|------|----------|
| < 0.90 | 差，结构失真明显 |
| 0.90 – 0.95 | 一般 |
| 0.95 – 0.98 | 良好 |
| 0.98 – 0.995 | 优秀 |
| > 0.995 | 极优，几乎无感知差异 |

**本报告值：0.9791** — 属于**良好**范围，处于 q1 最低质量等级下的合理水平。

---

### `bpp`（单位：bits per pixel，越低表示压缩率越高）

**每像素比特数**，直接反映压缩率。值越小说明压缩越彻底，但通常伴随质量下降。

| 范围 | 参考场景 |
|------|----------|
| > 2.0 bpp | 接近无损 / 高质量 JPEG |
| 0.5 – 2.0 bpp | JPEG 正常质量区间 |
| 0.1 – 0.5 bpp | 学习型编解码器中低质量等级 |
| 0.01 – 0.1 bpp | 超低比特率，极高压缩 |
| < 0.01 bpp | 极端压缩，仅适合内容极简单的图像 |

**本报告值：0.01213 bpp** — 属于**超低比特率**。这一数值之所以极低，是因为：
1. q1 是最低质量（最高压缩）等级；
2. 遥感影像切片中存在大量平滑区域（裸地、水体），本身熵值低、极易压缩；
3. `per_image` 显示差异极大：复杂纹理切片可达 0.04 bpp，纯净背景切片仅需 0.003 bpp。

评价 bpp 时必须结合 PSNR / MS-SSIM，高质量的模型应在**尽可能低的 bpp 下维持尽可能高的 PSNR**。

---

### `encoding_time` / `decoding_time`（单位：秒/图，越低越好）

单张图片的编码（压缩）与解码（还原）耗时，在 CPU 或 GPU 上均受批量大小、图像分辨率、模型复杂度影响。

**本报告值：编码 6.70 s / 解码 13.22 s**（CPU 单张，1024×1024 切片）。
学习型编解码器比传统编解码器（JPEG、WebP）慢 1~3 个数量级，速度是已知弱点。

---

## `per_image` 字段

与 `results` 相同的指标，但逐图记录，可用于：
- 识别哪些图块是"难例"（高 bpp 低 PSNR）；
- 分析模型在不同内容类型下的表现差异；
- 排查异常值（如某张图解码时间异常长）。

---

## 如何判断一个模型表现良好？

评价模型质量不能孤立看单一指标，应结合 **率失真曲线（Rate-Distortion curve）**，即在不同质量等级（q1 ~ q6）下的 bpp 与 PSNR / MS-SSIM 的关系曲线。

**参考基准（自然图像 Kodak 数据集，RGB，8-bit）：**

| bpp | PSNR 参考线（优秀模型） | JPEG 基线 |
|-----|------------------------|-----------|
| 0.12 | ~32 – 34 dB | ~28 – 30 dB |
| 0.25 | ~35 – 37 dB | ~31 – 33 dB |
| 0.50 | ~38 – 40 dB | ~34 – 36 dB |
| 1.00 | ~41 – 43 dB | ~37 – 39 dB |

> 注：本项目测试集为遥感影像（非 Kodak），内容特性不同，上述数值仅供量级参考，不可直接对比。

**综合判断原则：**
- 在相同 bpp 下，PSNR 和 MS-SSIM 越高越好；
- 在相同 PSNR 下，bpp 越低越好；
- 编解码时间在可接受范围内；
- 指标在不同内容图块上方差不宜过大（鲁棒性）。
